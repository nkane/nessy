# ADR 0005 — v0.5.0: VRC2/VRC4 mapper family, FME-7 + Sunsoft 5B audio, $2007 open-bus latch

- **Status:** Accepted
- **Release:** v0.5.0 (2026-05-24)
- **Theme:** Bring up the Konami VRC2/VRC4 mapper family and the Sunsoft FME-7 (with its YM2149-clone expansion audio), and pin the 2C02's shared-bus open-bus behaviour.

## D1 — One `VRC` type for the whole VRC2/VRC4 family (mappers 21/22/23/25)

- **Context:** Konami shipped the same VRC2/VRC4 silicon under four iNES mapper numbers (21, 22, 23, 25) plus sub-mapper variants. The register *classes* ($8000 PRG, $9000 mirroring/PRG-mode, $A000 PRG, $B000–$E000 CHR, $F000 IRQ) are identical across all of them; only the wiring of the two "sub-bits" inside each class differs by pinout. Modelling each mapper number as its own type would have duplicated the entire bank-switching surface four times.
- **Decision:** a single `cart.VRC` carries the shared wiring; a `vrcVariant` enum (VRC2a/b/c, VRC4a/b/f) selects the per-variant pin layout, and `subBits(addr)` is the only place the pinout differences live — it maps a register-write address to its 0..3 sub-index per variant. `NewVRC` resolves mapper number + sub-mapper to a variant (and the `isVRC4` flag) once at construction. The VRC2a outlier (mapper 22 routes only the upper 7 of 8 CHR bank bits) is handled by a single right-shift at CHR-read time rather than normalising stored bank values.
- **Consequences:** adding or correcting a pinout is a one-line `subBits` change, not a new mapper. PRG mode 1 (swap $8000/$C000) is VRC4-only and gated on `isVRC4`; VRC2 writes to the $F000 IRQ class fall through as no-ops. This is the template the later expansion-audio Konami mappers reuse (VRC6/VRC7 in ADR 0006).

## D2 — VRC4 CPU-clocked IRQ as a `cpu.Ticker`, VRC2 leaves it silent

- **Context:** VRC4 adds a software IRQ counter (the scanline counter Konami games lean on for raster splits) that VRC2 omits entirely. The counter has two modes — a scanline mode driven by a 341-CPU-cycle prescaler and a CPU mode that ticks every CPU cycle — plus an "enable after acknowledge" latch.
- **Decision:** `VRC.Tick(cycles)` (the `cpu.Ticker` hook from chippy, ADR 0002 D4) advances the counter; it early-returns unless `isVRC4 && irqEnable`. The $F000 register class decodes via `subBits` to latch-low / latch-high / control / acknowledge. On the $FF→reload edge the named `"vrc4"` IRQ source asserts through an `IRQSink`; acknowledge ($F000 sub 3) restores enable from the after-ack latch and clears the source.
- **Consequences:** VRC2 carts pay nothing for the IRQ machinery (the ticker bails immediately). Routing through a *named* IRQ source means multiple cart/APU IRQ lines share the CPU's single /IRQ surface without clobbering each other. The 341-cycle prescaler is the scanline approximation; sub-cycle raster precision is out of scope for v0.5.

## D3 — FME-7 (mapper 69) as a command/parameter mapper with a 16-bit CPU-clock IRQ

- **Context:** the Sunsoft FME-7 / 5B mapper (Gimmick!, Batman: Return of the Joker) uses a command-port / parameter-port pair ($8000 latches a 4-bit command, $A000 writes its parameter) covering eight 1 KiB CHR banks, four switchable 8 KiB PRG windows (the $6000 window sourceable from PRG-ROM *or* PRG-RAM), mirroring, and a 16-bit down-counter IRQ.
- **Decision:** a dedicated `cart.FME7` with the command/parameter decode in `applyParameter`, the fixed last-bank pinned at $E000, and a `Tick`-driven 16-bit counter that fires the named `"fme7"` IRQ source on underflow (pre-decrement: $0000 wraps to $FFFF and fires on that tick), keeps wrapping per nesdev, and acks on any command-13 write.
- **Consequences:** mirrors the VRC IRQ-sink + ticker shape from D2, so the CPU's /IRQ aggregation stays uniform across cart families. The $6000 PRG-RAM/ROM source selection (command 8 bits 7/6) is honoured for both read and write.

## D4 — Sunsoft 5B audio lives in `apu`, fed by the cart through a narrow sink

- **Context:** mapper 69 is two chips on one cartridge — the FME-7 bank-switcher and a Sunsoft 5B audio block (a YM2149 / PSG "SSG" clone: three square channels, per-channel amplitude, an envelope generator, a noise generator). The audio block is addressed through the *cart's* $C000 (address latch) / $E000 (data) port pair, but it is conceptually an APU peripheral, not a bank-switcher.
- **Decision:** keep the two halves decoupled. `apu.Sunsoft5B` owns the register file, tone/envelope state, and mixing; the cart exposes a one-method `cart.Sunsoft5BSink` (`Write(addr, v)`) and forwards $C000/$E000 writes to it via `FME7.SetAudioSink`. `cmd/nessy` wiring installs the APU's 5B chip into the cart after construction. With no sink wired, the cart silently drops the audio-port writes — matching a real FME-7-without-audio board.
- **Consequences:** the cart package never imports the apu package; the audio chip is testable in isolation. v0.5 ships a minimum-viable 5B: tone channels at the YM2149 /16 prescaler plus sawtooth (shape 0xE) / triangle (shape 0xA) envelopes — enough for Gimmick!. The noise LFSR and exotic envelope shapes are deliberately deferred. This is the first cartridge-chip audio path; it continues with VRC6 and the VRC7 OPLL FM synth in ADR 0006.

## D5 — $2007 PPU open-bus latch on the shared 2C02 data bus

- **Context:** the 2C02's external data bus is shared between CPU and PPU accesses, and it behaves as a capacitive latch that retains the last byte driven onto it. Write-only registers ($2000/$2001/$2003/$2005/$2006) read back this latch; $2002 drives only its top 3 bits and leaves the bottom 5 as latch; palette reads ($3F00–$3FFF) drive only 6 bits, leaving the top 2 as latch. `ppu_open_bus.nes` probes all of this.
- **Decision:** a single `openBus byte` field updated on every $2000–$2007 access. Writes to any register set it to the written byte. $2002 returns `(status & 0xE0) | (openBus & 0x1F)` and refreshes the latch's top 3 bits from status. $2007 non-palette reads return the read buffer and latch it; palette reads return `pal6 | (openBus & 0xC0)` and latch only the 6 palette bits, while still refilling the buffer from the nametable byte under the palette region. Write-only registers return the latch verbatim.
- **Consequences:** the latch is part of PPU save-state (`OpenBus` in the state struct) so it round-trips deterministically. We deliberately do **not** model per-bit DRAM-cell decay (~1 frame on silicon) — the latch holds indefinitely, which is correct for every shipping ROM that reads it and keeps captures deterministic. The vblank-flag read race and $2007 read buffering interact with this path; their cycle timing is owned by the master-clock model (chippy-side, ADR 0004).
